# AI-Augmented Project Management RAG Assistant

## Project Overview

This is a 30-day proof-of-concept project designed to explore how LLM, ML concepts, and Retrieval-Augmented Generation can support technical project management.

The assistant will focus on project reporting, risk visibility, blocker analysis, QA readiness, and historical issue retrieval.

## Problem Statement

Project knowledge is often scattered across sprint notes, QA defect reports, meeting notes, risk logs, and stakeholder updates. This makes it difficult for a Project Manager to quickly identify recurring blockers, summarize risks, and prepare grounded weekly reports.

## Target Bottleneck

The main bottleneck this project will solve is:

Mapping QA defects, sprint notes, blockers, and historical project issues into a searchable AI knowledge base that can support faster and more accurate project reporting.

## 30-Day MVP Scope

The MVP will:

1. Ingest sample project notes, QA defects, meeting notes, and risk logs.
2. Convert unstructured PM text into structured data.
3. Store project knowledge using PostgreSQL and pgvector concepts.
4. Use embeddings and semantic search to retrieve relevant project context.
5. Generate grounded PM outputs such as status reports, risk summaries, blockers, RAID logs, and QA readiness checks.
6. Require human review before any output is treated as final.

## Out of Scope

The first 30-day version will not include:

1. Production deployment.
2. Real confidential company data.
3. Automatic updates to Jira, Azure Boards, or company systems.
4. Autonomous decision-making.
5. Direct stakeholder email sending.

## Target Users

1. Technical Project Manager
2. QA Lead
3. Development Lead
4. Delivery Manager
5. Stakeholders who need project visibility

## Success Criteria

By Day 30, the prototype should be able to demonstrate:

1. Document ingestion
2. Structured extraction
3. Basic semantic retrieval
4. Grounded weekly reporting
5. Risk and blocker identification
6. QA readiness summary
7. Human-in-the-loop review
8. Evaluation scorecard
